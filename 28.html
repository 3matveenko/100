В Java инкапсуляция обеспечивается с использованием модификаторов доступа и геттеров/сеттеров. Основные элементы, отвечающие за инкапсуляцию в Java:

  1. Модификаторы доступа:
    • private: Переменные и методы, объявленные с модификатором private, доступны только внутри того же класса.
    • default (package-private): Если не указан модификатор доступа (по умолчанию), то элементы видны только внутри того же пакета.
    • protected: Доступно внутри того же пакета и для подклассов даже из других пакетов.
    • public: Доступно из любого места.
java
public class MyClass {
    private int myPrivateVariable;
    int myDefaultVariable;
    protected int myProtectedVariable;
    public int myPublicVariable;

    private void myPrivateMethod() {
        // Логика метода
    }

    // Геттер и сеттер для private переменной
    public int getMyPrivateVariable() {
        return myPrivateVariable;
    }

    public void setMyPrivateVariable(int value) {
        this.myPrivateVariable = value;
    }
}
  1. Геттеры и сеттеры (getter and setter): Используются для получения и установки значений приватных полей класса.
java
public class MyClass {
    private int myPrivateVariable;

    // Геттер
    public int getMyPrivateVariable() {
        return myPrivateVariable;
    }

    // Сеттер
    public void setMyPrivateVariable(int value) {
        this.myPrivateVariable = value;
    }
}

Эти механизмы помогают скрыть детали реализации от внешнего кода и управлять доступом к членам класса, обеспечивая тем самым инкапсуляцию.
