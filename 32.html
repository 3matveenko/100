В программировании, в том числе и на языках Java, PHP, и JavaScript, термин "перегрузка метода" (method overloading) относится к возможности определения нескольких методов с одним и тем же именем в одном классе или объекте, но с разными параметрами. В случае мобильной разработки на языке Kotlin, аналогичный концепт применяется к функциям.

Пример на Java:

java
public class Example {
    // Перегрузка метода с разными типами параметров
    public void printInfo(String message) {
        System.out.println("Сообщение: " + message);
    }

    public void printInfo(int number) {
        System.out.println("Число: " + number);
    }

    // Перегрузка метода с разным количеством параметров
    public void printInfo(String message, int number) {
        System.out.println("Сообщение: " + message + ", Число: " + number);
    }
}

Пример на PHP:

php
class Example {
    // Перегрузка метода с разными типами параметров
    public function printInfo($message) {
        echo "Сообщение: " . $message . "\n";
    }

    public function printInfo($number) {
        echo "Число: " . $number . "\n";
    }

    // Перегрузка метода с разным количеством параметров
    public function printInfo($message, $number) {
        echo "Сообщение: " . $message . ", Число: " . $number . "\n";
    }
}

Пример на JavaScript:

javascript
class Example {
    // Перегрузка метода с разными типами параметров
    printInfo(arg) {
        if (typeof arg === 'string') {
            console.log("Сообщение: " + arg);
        } else if (typeof arg === 'number') {
            console.log("Число: " + arg);
        }
    }

    // Перегрузка метода с разным количеством параметров
    printInfo(message, number) {
        console.log("Сообщение: " + message + ", Число: " + number);
    }
}

Важно отметить, что перегрузка методов обеспечивает более гибкую работу с методами, позволяя использовать одно и то же имя метода для различных сценариев использования.
